# qvra repo bootstrap checklist

Use this when a new repo is created.

## Required

- exact repo role
- exact neighboring boundaries
- README with route-first structure
- `.gitignore`
- remote configured
- clean commit titles
- at least one real surface or clearly bounded public foundation
- route links to adjacent repos where relevant

## Before first publish

Confirm:
- why this is a repo and not a directory in another repo
- what traffic role it serves
- what it must never become
- whether it strengthens route density or only adds structure

## Before adding more files

Ask:
- is this increasing utility, comparison, experimentation, visible proof, or recurrence?
- or is it only making the topology feel complete?

If it is only topology satisfaction, stop.
