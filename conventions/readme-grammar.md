# qvra README grammar

Public READMEs in `qvra` should privilege result over explanation.

## Default order

1. repository name
2. one-line role statement
3. what belongs here
4. what does not belong here
5. current role in qvra
6. live routes to neighboring repos
7. first live artifact or current notable surface
8. status

## Rules

- result before doctrine
- route density before abstraction
- fast legibility before total explanation
- clear repo boundary before broad ambition

## Avoid

- manifesto openings
- lore requirements
- abstract self-importance
- screenshots without route
- claims without runnable or inspectable proof

## Tone

- exact
- sharp
- non-ceremonial
- high-signal
- low-friction
