# meta

Pattern engine for `qvra`.

`meta` carries the shared standards that let the field grow without turning generic, noisy, or structurally confused.

It exists to make new repos feel natively `qvra` without forcing the anchors to explain the operating grammar themselves.

## What belongs here

- issue templates
- pull request structure
- label taxonomy
- README grammar
- repo bootstrap rules
- commit title rules
- shared field conventions

## What does not belong here

`meta` is not:
- user-facing product documentation
- a manifesto archive
- a junk drawer for unrelated utilities
- a substitute for live repos

## Current role in qvra

`meta` is part of the structural spine.
It keeps the field coherent while the anchors stay loud.
